Workout & history folders
History folder
Workout library (.zwo)
VeloDrive stores its data in two folders:
  • History folder – each workout you complete is exported here as a FIT file with power, HR, cadence, and metadata.
  • Workout library folder – this points to the folder where your .zwo workouts will be stored.
If you are just getting started, we recommend the following layout: 
    ~/Documents/velodrive/
    ├── history/      ← exported .fit files from completed workouts
    └── workouts/     ← your .zwo workout files
Workout behaviour
Functional Threshold Power (FTP)
W
FTP is your sustainable power for roughly a 1-hour effort. VeloDrive uses it to:
  • Convert relative workout segments (%FTP) into watts.
  • Estimate IF (intensity factor), TSS, and energy.
  • Colour-code zones in the workout chart and HUD.
If you’re unsure, start with a rough guess (e.g. last known FTP or outdoor 20–30 minute best power × 0.95) and adjust over time.
Interval sounds
When sounds are enabled:
  • Short beep pattern a few seconds before a big interval change.
  • More dramatic sound before very hard efforts.
  • Countdown & overlay when starting a workout.
This helps you prepare for intensity changes without staring at the screen.
Environment checks
Web Bluetooth
Web Bluetooth is what allows VeloDrive to connect directly to your trainer and heart-rate monitor from the browser. Some browsers or configurations don’t support it yet, which is why we recommend recent Chromium-based browsers and, if needed, enabling the Bluetooth flags in chrome://flags.

In Chrome desktop, open a new tab and navigate to:
  • chrome://flags/#enable-web-bluetooth
  • chrome://flags/#enable-web-bluetooth-new-permissions-backend
Set them to Enabled, restart the browser, and then reload VeloDrive.

Web Bluetooth is only guaranteed to work in Google Chrome on Linux, Windows, macOS, and Android. Some Chromium-based browsers (for example Vivaldi) do not support Bluetooth at all, and iOS browsers currently do not support Web Bluetooth.
